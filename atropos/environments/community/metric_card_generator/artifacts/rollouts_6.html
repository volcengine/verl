Use Case 1: Chat Support - Resolving User Inquiry About Recent Transaction

Title: User Queries Recent Transaction via Chat Support

User Goals & Objectives:
- Obtain detailed information about a recent transaction
- Clarify any discrepancies or errors
- Get guidance on dispute process if needed

Step-by-Step Flow:

  1. User opens the mobile banking app and navigates to chat support.
  2. User initiates a chat session and types: "I want details about my last transaction."
  3. System recognizes intent and presents recent transaction history.
  4. User selects a specific transaction or types transaction details (e.g., date, amount).
  5. System displays comprehensive details (merchant, date, amount, location).
  6. User inquires about a discrepancy: "Why was I charged twice?"
  7. System prompts user to specify which transaction to dispute.
  8. User confirms and requests to dispute the transaction.
  9. System guides user through dispute process or escalates to a live agent if needed.
  10. User receives confirmation of dispute initiation, with estimated resolution time.

Expected Outcomes:
- User successfully retrieves transaction info.
- Dispute is recorded and processed.
- Support interaction is logged for future reference.

Edge Cases / Alternative Paths:
- User's transaction not found (older transaction, incorrect details). System suggests alternative search criteria or escalates to live agent.
- Chatbot cannot understand user's query. Option to escalate to human support.
- User disconnects during the process; system saves chat context for future.

Use Case 2: Innovative Payment - Peer-to-Peer Money Transfer Using QR Code

Title: User Sends Money to Friend via QR Code

User Goals & Objectives:
- Quickly transfer funds to a trusted contact
- Use contactless method for convenience and security

Step-by-Step Flow:

  1. User logs into the mobile banking app.
  2. User navigates to "Transfer" > "Send via QR Code."
  3. System displays QR code representing user’s payment info.
  4. Friend opens their app, scans the QR code.
  5. System verifies the recipient's identity and confirms the amount (or prompts user to specify amount).
  6. User enters the transfer amount, reviews details.
  7. User authorizes the transaction via biometric authentication or PIN.
  8. System processes payment, updates both accounts, and sends confirmation notifications.
  9. Both users receive transaction confirmation via app and optional email/SMS.

Expected Outcomes:
- Funds are transferred securely and promptly.
- Transaction is reflected in both users’ account histories.

Edge Cases / Alternative Paths:
- QR code expired or invalid (e.g., scanned after expiry). System prompts to generate a new code.
- Insufficient funds in sender's account. System notifies and offers options.
- User cancels before completing payment.
- Connectivity issues during scanning or processing.
